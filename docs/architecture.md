# LovTrip 系统架构设计

> Architecture Design Document

---

## 整体架构

LovTrip 采用**分层架构**（Layered Architecture），将系统分为表现层、组件层、服务层和配置层。

```
┌─────────────────────────────────────────────────────────┐
│                    用户浏览器 (Browser)                    │
├─────────────────────────────────────────────────────────┤
│              Streamlit Server (Python)                    │
│  ┌───────────────────────────────────────────────────┐  │
│  │                  UI 表现层 (streamlit_app.py)       │  │
│  │  ┌──────────┐ ┌──────────┐ ┌──────────────────┐  │  │
│  │  │  Hero    │ │ Sidebar  │ │ Main (Map+Result) │  │  │
│  │  │  Section │ │ Controls │ │      Area         │  │  │
│  │  └──────────┘ └──────────┘ └──────────────────┘  │  │
│  ├───────────────────────────────────────────────────┤  │
│  │              组件层 (components/)                   │  │
│  │  ┌──────────┐ ┌──────────┐ ┌──────────────────┐  │  │
│  │  │ styles   │ │   map    │ │       poi        │  │  │
│  │  │ (CSS)    │ │ (Leaflet)│ │   (POI Cards)    │  │  │
│  │  └──────────┘ └──────────┘ └──────────────────┘  │  │
│  ├───────────────────────────────────────────────────┤  │
│  │              服务层 (services/)                     │  │
│  │  ┌──────────┐ ┌──────────┐ ┌──────────────────┐  │  │
│  │  │ weather  │ │baidu_map │ │   OpenAI Client  │  │  │
│  │  │ (天气)   │ │(地图/POI)│ │   (LLM 调用)    │  │  │
│  │  └──────────┘ └──────────┘ └──────────────────┘  │  │
│  ├───────────────────────────────────────────────────┤  │
│  │              配置层 (config/)                       │  │
│  │  ┌──────────────────────────────────────────────┐  │  │
│  │  │  API Keys / System Prompt / 常量定义         │  │  │
│  │  └──────────────────────────────────────────────┘  │  │
│  └───────────────────────────────────────────────────┘  │
└─────────────────────────────────────────────────────────┘
                          │
          ┌───────────────┼───────────────┐
          ▼               ▼               ▼
    ┌──────────┐   ┌───────────┐   ┌──────────┐
    │  TokenHub │   │ 百度地图  │   │Open-Meteo│
    │   LLM    │   │   API     │   │  Weather │
    └──────────┘   └───────────┘   └──────────┘
```

## 分层说明

### 1. 配置层 (`config/`)

**职责**：统一管理所有配置项

- API Key 加载（优先 Streamlit Secrets，回退环境变量）
- LLM 参数（模型名称、Base URL）
- System Prompt 定义
- 第三方 API Key（百度地图、Pexels）

**设计原则**：配置与代码分离，所有 Key 不硬编码。

### 2. 服务层 (`services/`)

**职责**：封装所有外部 API 调用，提供纯函数接口

| 模块 | 功能 | 外部依赖 |
|------|------|----------|
| `baidu_map.py` | 地理编码 + 地点搜索 | 百度地图 API |
| `weather.py` | 实时天气查询 | Open-Meteo API + 百度地图（坐标） |

**设计原则**：
- 纯函数，无副作用
- 统一错误处理（异常返回 None/[]）
- 不依赖 Streamlit 运行时

### 3. 组件层 (`components/`)

**职责**：可复用的 HTML/CSS/JS 渲染组件

| 模块 | 功能 | 输出 |
|------|------|------|
| `styles.py` | LovTrip 主题 CSS | CSS 字符串 |
| `map.py` | Leaflet 交互地图 | HTML 字符串 |
| `poi.py` | POI 卡片网格 + Pexels 图片 | HTML 字符串 |

**设计原则**：
- 返回 HTML 字符串，由 Streamlit 渲染
- 可独立测试（不依赖 Streamlit 运行时）

### 4. UI 表现层 (`streamlit_app.py`)

**职责**：
- Streamlit 页面骨架
- Session State 管理
- 用户交互流程编排
- 各层组件的协调调用

## 数据流

```
用户输入目的地 & 偏好
          │
          ▼
    ┌─────────────┐
    │ 配置层加载   │ ← API Keys, System Prompt
    │ API Keys    │
    └─────────────┘
          │
          ▼
    ┌─────────────┐
    │ 天气查询     │ → Open-Meteo API
    │ (同步)      │
    └─────────────┘
          │
          ▼
    ┌─────────────┐
    │ 地图渲染     │ → 百度地图 API (geocode + POI search)
    │ + POI 展示  │ → Leaflet 前端渲染
    └─────────────┘
          │
          ▼ (用户点击"生成行程")
    ┌─────────────┐
    │ LLM 调用     │ → TokenHub API
    │ (注入天气)   │ → System Prompt + User Input + Weather Context
    └─────────────┘
          │
          ▼
    ┌─────────────┐
    │ 结果渲染     │ → Markdown 展示
    │ + 下载按钮   │ → TXT 导出
    └─────────────┘
```

## 技术选型理由

| 技术 | 选型理由 |
|------|----------|
| **Streamlit** | Python 原生 Web 框架，零前端代码，快速迭代 |
| **Leaflet** | 开源免费，无需 API Key，全球覆盖，替代百度地图 JS SDK |
| **Open-Meteo** | 免费开源天气 API，无需 Key，替代付费天气服务 |
| **TokenHub / Kimi** | 国内可用 LLM，支持 OpenAI 兼容接口 |
| **Pexels** | 高质量免费图片 API，丰富 POI 视觉体验 |
| **百度地图 API** | 国内 POI 数据最全面，地理编码精度高 |

## 安全设计

```
┌────────────────────────────────────┐
│         API Key 加载策略            │
├────────────────────────────────────┤
│  1. st.secrets (Streamlit Cloud)   │
│     ↓ 失败                          │
│  2. os.environ (本地 .env)          │
│     ↓ 均失败                        │
│  3. st.error() + st.stop()         │
└────────────────────────────────────┘
```

- `.streamlit/secrets.toml` → gitignored
- `.env` → gitignored
- `.env.example` → 提交（模板，无真实值）
- Git 历史中无任何 API Key 提交记录

## 扩展性设计

### 添加新 API 服务

1. 在 `services/` 创建新模块
2. 在 `config/` 添加对应 Key 配置
3. 在 `streamlit_app.py` 中导入使用

### 添加新 UI 组件

1. 在 `components/` 创建新模块
2. 导出返回 HTML/字符串的纯函数
3. 在 `streamlit_app.py` 中调用渲染
